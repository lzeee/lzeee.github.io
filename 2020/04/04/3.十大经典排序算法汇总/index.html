十大经典排序算法汇总

Posted on 2020-04-04 In 学习 Disqus:
Symbols count in article: 9.3k Reading time ≈ 8 mins.
面试能bug free写出来吗？

0.概述

有很多文章汇总了这十种排序算法,内容结构大体一致。我觉得只看不行,还是实现对比一下.一般来说,考虑排序算法时,会考虑他们的时间复杂度,空间复杂度以及稳定性,在时间复杂度上,又区分了一般情况,最好情况以及最差情况.此外,许多文章提到了排序算法是否稳定,这里稳定指的是两个相等的元素在排序是否改变原有的顺序.

0.1 分类

下面是一张排序算法的分类图,比较排序是说通过对要素间的对比来决定次序,时间复杂度不可能突然O(NlogN),所以也称为非线性时间比较排序,另一类非比较排序的方法通过一定的技巧,可以不通过比较要素来完成排序,这类方法突破了前者的时间下限,可以在线性时间内完成要素的排序.

关于不同算法的复杂度分析想要放在最后,因为我认为关键不是记住,而是理解了这些算法,以及为什么有这样的复杂度.

1.具体算法

本文以下所有算法均由不那么标准的C++实现,并且排序目标均为升序排列.

1.1 冒泡排序

  • 算法思想
    冒泡排序大概是知道的人最多的排序算法,毕竟文科生都开始上计算概论了,挠头.这个算法的核心就在于相邻元素之间两两对比,符合要求的挪一挪,这样从左到右走一遍,最符合要求的那个元素就被挪到了最右边,过一遍有一个元素被放到了合适的位置,所以要过n遍.总的来说,名字有趣但没太大意思.

  • 具体实现

  • 复杂度分析
    空间复杂度是O(1),不需要额外的空间.时间复杂度平均为O(N^2),最好O(N),最差O(N^2).什么情况最好呢?当数据本身就是升序排列时,并且用一个量来标记是否发生过挪动,未挪动过就终止,这样就可以以O(N)的速度完成。最差就是完全反序的时候，每次对比都需要挪动。冒泡排序是稳定的。

1.2 选择排序

  • 算法思想
    选择排序是走一趟找一个最小值。每走一趟就找到了一个最小的，将最小值放到前面排好序部分的末尾，走N-1趟序列就变得有序了。和冒泡排序一样是双重循环。

  • 具体实现

  • 复杂度分析
    选择排序的复杂的是O(N^2)，最差最好和平均情况都是一样的，这个算法是不稳定的，因为交换可能会打乱原有相同数字之间的顺序。其实究竟是否稳定和实现方式有关。

1.3 插入排序

  • 算法思想
    其实我一直分不太清插入排序和选择排序，可以想选择排序是一次选一个最小的，插入排序则是从头开始，把一个数字往前插，直到找到合适的位置，往前插的过程需要不断交换，过程就像打牌时，拿到小牌往前插的感觉，有趣。

  • 具体实现

  • 复杂度分析
    插入排序的复杂度是O(N^2)，最好情况下如果本身是有序的，复杂度可以到O(N)，这个算法是稳定的，其实是看挪动的条件，如果相等时也挪那就不稳定了，当然相等时没必要挪。

1.4 希尔排序

  • 算法思想
    希尔排序和插入排序的基本思想一致，但是插入时的对比是跳跃着比，跳跃的步长逐渐减小，为1时就是普通的插入排序。可以理解为先使序列变得基本有序，在有序条件下的插入排序是更快的。

  • 具体实现

  • 复杂度分析
    此方法复杂度的分析似乎不够明确，一般认为是O(NlogN)。希尔排序是不稳定的。

1.5 归并排序

  • 算法思想
    归并排序的思想是典型的分治，分而治之，将有序序列不断合并，最后的序列是有序的。二路归并就是分成两块，按道理可以分成许多块。

  • 具体实现

  • 复杂度分析
    复杂度是O(NlogN),推导方法是用T(N) = T(N/2) + O(N)，到最后累加求出。归并排序是稳定的，其实就是一个个小区间逐渐调整然后融合，不会改变相对位置。

1.6 快速排序

  • 算法思想
    快速排序的思想，在之前看过的《啊哈！算法》中有提到过，核心就是在给一个基准值找确切位置的时候，使得剩下的左右两部分变得整体有序，也就是说左边的元素都小于基准值而右边的元素都大于基准值，这步关键操作经常被称为partition。

  • 具体实现

  • 复杂度分析
    空间复杂度主要考虑的是递归调用消耗的空间，O(logN)。时间复杂度平均情况是O(NlogN),最差情况是O(N^2)，什么时候出现最差情况？当完全反序时，每次递归调用的一个区间是空的，另一个区间长度逐渐减1，每次partition操作需要把序列全过一遍，最后就是N方了。快排是不稳定的。

1.7 堆排序

  • 算法思想
    堆排序实际上是利用堆进行排序的算法，构造一个最大堆/最小堆，依次将根节点取出并调整堆，最后取出的序列就是有序的。该方法听起来容易理解，难点在于堆的建立。有一个技巧，可以用数组来存堆，并且快速建堆。

  • 具体实现

  • 复杂度分析
    堆排序的复杂度主要在于堆的调整，用技巧快速建堆的复杂度是O(N)，调整堆的时间复杂度为O(logN)，找N次可以看成是O(NlogN)。堆排序算法是不稳定的。

1.8 计数排序

  • 算法思想
    这最后的的三种算法，都使用到了额外的空间，从而加快了排序的速度，可以说是空间换时间的考虑。而计数排序这种非常之常见的技巧，也即使用数值本身作为map的key值，其实还是蛮有意思的。

  • 具体实现

  • 复杂度分析
    计数排序时间复杂度为线性，空间复杂度为O(N)，可以是稳定的，和写法有关。

1.9 桶排序

  • 算法思想
    桶排序和计数排序比起来，在我看来二者的关系是后者是前者的特例，计数排序的桶的大小为1，而且映射函数为数值本身。桶排序整体流程是，先将数分到不同的桶内，桶内再继续排序，最后按桶的顺序排列就得到了有序的序列。

  • 具体实现

  • 复杂度分析
    桶排序的时间复杂度影响因素很多，最好情况下是线性，最差的情况下为O(N^2)，最差情况比如说所有要素在一个桶内而且桶内使用冒泡排序？桶排序是稳定的。

1.10 基数排序

  • 算法思想
    基数排序似乎是我在系统总结十大排序之前没有见过的一个排序算法。不得不说其中包含的思考真的很巧妙。对序列中的每一个元素，根据从低到高的位数上的数字依次排序，最后就得到了有序序列。

  • 具体实现

  • 复杂度分析
    基数排序的空间复杂度为常量，时间复杂度主要是看最大值的位数k，随着位数的增加而增大。基数排序是稳定的。

2.总结

2.1 常见排序算法复杂度

算法名称 空间复杂度 时间复杂度 最差 最好 稳定性*
冒泡排序 O(1) O(N^2) O(N^2) O(N) 稳定
选择排序 O(1) O(N^2) O(N^2) O(N^2) 不稳定
插入排序 O(1) O(N^2) O(N^2) O(N) 稳定
希尔排序 O(1) O(NlogN) O(NlogN) O(NlogN) 不稳定
归并排序 O(N) O(NlogN) O(NlogN) O(NlogN) 稳定
快速排序 O(logN) O(NlogN) O(NlogN) O(NlogN) 不稳定
堆排序 O(1) O(NlogN) O(NlogN) O(NlogN) 不稳定
计数排序 O(k*) O(N+k) O(N+k) O(N+k) 稳定
桶排序 O(N+k) O(N+k) O(N^2) O(N+k) 稳定
基数排序 O(N+k) O(N*k) O(N*k) O(N*k) 稳定

*稳定与否和实现方式有关
*k表示桶的数量

2.2 思考

看起来挺简单的排序算法，写起来其实还是挺花时间的，并没有想象中的简单。实际动手编写的过程也是一个加深理解的过程，我觉得以后有必要多总结。学编程很多年了，接触排序到现在也已经6年了，回过头来看，这是第一次系统的总结一类东西，还是需要多多反思，努力提高自己吧。
